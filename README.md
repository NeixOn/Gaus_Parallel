## Инструкция по запуску на linux
sudo apt update

sudo apt install gcc openmpi-bin libopenmpi-dev -y

mpicc -std=c11 -O2 -Wall -o gauss main.c gauss_1d.c gauss_2d.c gauss_1d_parallel.c gauss_2d_parallel.c test.c -lm

./gauss 2000 0

./gauss 2000 1

./gauss 2000 2


первое значение - размер матрицы, второе значение - выбор режима

0 - Параллельный метод гаусса для матрицы реализованной при помощи двумерного массива

1 - Параллелный метод Гаусса для матрицы реализованной при помощи одномерного массива

2 - Последовательный метод Гаусса для обоих видов реализаций матрицы

### Сравнение ускорения (Speedup) в Release-сборках

Speedup = T<sub>seq</sub> / T<sub>par</sub> (значение > 1 означает ускорение)

| Размер | MinGW Release | Visual Studio Release |
|--------|---------------|-----------------------|
| 1000 | 0.63× | 0.81× |
| 4000 | 0.65× | 0.68× |

> Параллельная версия уступает последовательной в Release-сборках из-за агрессивных оптимизаций компилятора (векторизация, развертка циклов), которые не могут быть применены к параллельному коду.

### Выводы

1. **Последовательная версия** эффективнее в Release-сборках (ускорение 3.5–5× относительно Debug)
2. **Параллельная версия** показывает ожидаемое ускорение только в Debug-сборках (1.5–2.2×)
3. **1D-представление** (плоский массив) незначительно быстрее 2D (массив указателей) за счет лучшей локальности данных
4. **Microsoft MPI** на Windows вносит дополнительные накладные расходы, снижая эффективность параллелизации
5. Для больших матриц (>5000) рекомендуется использовать Linux с OpenMPI или специализированные библиотеки (BLAS/LAPACK)

## Команды сборки
# Конфигурация CMake (автоматически)
cmake -G "MinGW Makefiles" -DCMAKE_BUILD_TYPE=Debug -B cmake-build-debug
cmake -G "MinGW Makefiles" -DCMAKE_BUILD_TYPE=Release -B cmake-build-release
# Фактическая компиляция (что видно в логах)
gcc -g -O0 -c gauss_2d_parallel.c -o gauss_2d_parallel.obj    # Debug
gcc -O3 -c gauss_2d_parallel.c -o gauss_2d_parallel.obj        # Release
gcc -o Gaus_Parallel.exe *.obj -lmsmpi                          # Линковка

# Конфигурация CMake
cmake -G "Visual Studio 17 2022" -DCMAKE_BUILD_TYPE=Debug -B cmake-build-debugvs
cmake -G "Visual Studio 17 2022" -DCMAKE_BUILD_TYPE=Release -B cmake-build-releasevs
# Фактическая компиляция
cl /Zi /Od /c gauss_2d_parallel.c           # DebugVS (но -O0 не поддерживается, отсюда warning)
cl /O2 /c gauss_2d_parallel.c               # ReleaseVS (но -O3 не поддерживается, отсюда warning)
link /OUT:Gaus_Parallel.exe *.obj msmpi.lib # Линковка
